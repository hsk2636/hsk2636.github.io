핵심 코드 / 패턴 예시

1) 결측치(Missing Values) 처리
import pandas as pd

# 예시 데이터프레임
df = pd.DataFrame({
    "age": [23, 25, None, 29],
    "income": [3000, None, 4200, 3900],
    "city": ["Seoul", "Busan", "Seoul", None]
})

# 1) 수치형 컬럼 평균으로 채우기
df["age"] = df["age"].fillna(df["age"].mean())

# 2) income 결측치는 0으로 채우기 (비즈니스 규칙에 따라)
df["income"] = df["income"].fillna(0)

# 3) city 는 'Unknown' 으로 채우기
df["city"] = df["city"].fillna("Unknown")

단순히 fillna만 쓰는 게 아니라, “어떤 컬럼은 평균으로”, “어떤 컬럼은 0이나 Unknown으로” 같은 비즈니스 규칙 기반 전처리까지 고민했다는 걸 어필할 수 있어.

2) Label Encoding / One-Hot Encoding
from sklearn.preprocessing import LabelEncoder
import pandas as pd

df = pd.DataFrame({
    "color": ["red", "blue", "green", "blue"],
    "size": ["S", "M", "L", "M"]
})

# 1) Label Encoding
le = LabelEncoder()
df["color_label"] = le.fit_transform(df["color"])

# 2) One-Hot Encoding (Pandas)
df_onehot = pd.get_dummies(df, columns=["size"], prefix="size")

print(df)
print(df_onehot)

Label Encoding vs One-Hot Encoding 각각의 특징, “순서가 없는 범주형 변수에 Label Encoding만 쓰면 모델이 순서를 오해할 수 있다” 같은 주의점을 함께 정리해 두면 훨씬 설득력 있는 포트폴리오가 돼.

3) SMOTE로 불균형 데이터 처리
from imblearn.over_sampling import SMOTE
from collections import Counter
import pandas as pd

# X : 독립변수, y : 종속변수(레이블)
X = df.drop("target", axis=1)
y = df["target"]

print("Before:", Counter(y))

smote = SMOTE(random_state=42)
X_res, y_res = smote.fit_resample(X, y)

print("After :", Counter(y_res))

단순히 “SMOTE 썼다”가 아니라 전/후 클래스 분포(Before/After)를 같이 출력해서 불균형이 얼마나 완화됐는지 보여주는 게 핵심이야.

전처리 + 모델링 파이프라인 예시

여러 전처리 단계를 파이프라인(Pipeline)으로 묶어서 “전처리 + 모델 학습”을 한 번에 돌리는 예제도 추가할 수 있어. 

from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

numeric_features = ["age", "income"]

pipeline = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="mean")),
    ("scaler", StandardScaler()),
    ("model", LogisticRegression())
])

pipeline.fit(X_train[numeric_features], y_train)
score = pipeline.score(X_test[numeric_features], y_test)

print("Test Accuracy:", score)

이런 식으로 “실제 모델링까지 고려한 전처리 흐름”을 보여주면 단순 실습을 넘어서 실무 ML 파이프라인 감각까지 어필할 수 있어.
